     Plan: Sistem Shift 24 Jam (Fleksibel Multi-Roster per Hari)

     Context

     Saat ini sistem shift hanya mendukung 1 jadwal per hari per template, dan
     menolak shift lintas tengah malam (validasi work_end_time > 
     work_start_time). User ingin:

     - Kantor 24 jam bisa dibagi jadi 3 roster × 8 jam (06:00–14:00,
     14:00–22:00, 22:00–06:00)
     - Kantor 16 jam → 2 roster × 8 jam
     - Fleksibel — jumlah roster dan durasi bisa diatur bebas
     - Template bisa diaktifkan/nonaktifkan (sudah ada toggle endpoint)
     - Sistem presensi (check-in/out, overtime, auto-checkout) harus berjalan
     benar

     Masalah Teknis yang Harus Diselesaikan

     1. Cross-midnight shift (22:00–06:00)

     - Validasi validateSchedules() saat ini MENOLAK end <= start → harus
     diizinkan
     - checkIn() → determineStatus() menentukan telat berdasarkan
     work_start_time. Untuk shift 22:00–06:00, karyawan check-in jam 22:15 harus
     dinilai berdasarkan 22:00, bukan 06:00
     - checkOut() → calculateOvertime() menghitung lembur setelah work_end_time.
     Untuk shift 22:00–06:00, jam pulang 06:00 jatuh di tanggal berikutnya
     - AutoCheckoutCommand → reminder_at dan auto_checkout_at harus berdasarkan
     jam pulang di tanggal berikutnya
     - resolveSchedule() → shift 22:00–06:00 yang dimulai hari Senin harus tetap
     berlaku saat check-out hari Selasa pagi. Perlu is_cross_day flag

     2. Multi-roster per hari

     Saat ini 1 shift template = 1 jadwal per hari (7 baris shift_schedules).
     Untuk 3 roster per hari, ada 2 pendekatan:

     Pendekatan yang dipilih: Setiap roster = 1 template shift terpisah. HRD
     buat 3 template: "Pagi 06–14", "Siang 14–22", "Malam 22–06" dan assign
     karyawan ke masing-masing. Ini sesuai desain yang sudah ada (assign
     karyawan ke shift template), tanpa mengubah struktur DB shift_schedules.

     Perubahan yang Diperlukan

     A. Migration: tambah is_cross_day ke shift_schedules

     ALTER TABLE shift_schedules ADD COLUMN is_cross_day BOOLEAN DEFAULT false
     AFTER is_off;
     is_cross_day = true → work_end_time jatuh di tanggal berikutnya (shift
     malam 22:00–06:00).

     B. Backend — ShiftController.php

     1. validateSchedules() — hapus penolakan end <= start, ganti dengan: jika
     end <= start → wajib set is_cross_day = true di payload (otomatis di
     backend)
     2. store() & update() — tambah validasi is_cross_day di schedule input
     (sometimes|boolean)
     3. syncSchedules() — simpan is_cross_day
     4. resolveSchedule() — return juga is_cross_day di array hasil. Tambah
     logika: jika hari ini tidak ada jadwal shift yang cocok, cek jadwal hari 
     kemarin — jika is_cross_day = true dan sekarang masih dalam window
     work_start_time(kemarin) s/d work_end_time(hari ini), pakai jadwal kemarin

     C. Backend — AttendanceController.php

     1. determineStatus() — jam masuk shift cross-day tetap pakai
     work_start_time hari itu (contoh: 22:00). Karyawan telat jika check-in
     setelah 22:00 + toleransi
     2. calculateOvertime() — jam pulang cross-day: Carbon::parse(date+1 . ' ' .
     work_end_time) (tanggal berikutnya). Lembur dihitung setelah jam itu
     3. checkIn() — reminder_at dan auto_checkout_at harus pakai tanggal
     berikutnya jika is_cross_day
     4. checkOut() — work_minutes sudah benar (diff check_in s/d check_out).
     Overtime perlu aware cross-day

     D. Backend — AutoCheckoutCommand.php

     - $jamPulang untuk shift cross-day harus jadi date+1 . ' ' . work_end_time
     - Grace period dan reminder dihitung dari jam pulang di tanggal berikutnya

     E. Model ShiftSchedule.php

     - Tambah is_cross_day ke $fillable dan casts

     F. Frontend Web — ShiftManagement.tsx

     1. ShiftFormModal — di editor jadwal per hari:
       - Jika work_end_time <= work_start_time, otomatis set is_cross_day = true
     dan tampilkan badge "(+1 hari)"
       - Hapus pesan error "shift lintas tengah malam belum didukung"
       - Tampilkan info durasi jam kerja per hari (contoh: "8 jam")
     2. Template card — tampilkan badge +1 di mini jadwal untuk hari cross-day
     3. Payload — kirim is_cross_day per schedule entry

     G. Frontend Web — SettingsManagement.tsx

     Tidak perlu diubah — work_days kantor sudah ditambahkan di task sebelumnya.
